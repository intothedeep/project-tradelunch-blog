# blog_agent

Multi-agent CLI that turns a local markdown file into a published blog post.
Built with **LangGraph** + an LLM backend (**Qwen3 8B via Ollama** by default,
optional OpenAI / Anthropic). Python, managed by **uv**.

## What it can do

- Parse a markdown post (YAML frontmatter + body) and extract metadata.
- Generate **slug, tags, category, and summary** from the content via LLM
  (frontmatter tags/description are ignored — always LLM-derived).
- Optimize the thumbnail to the **Open Graph 1200×630** standard (Pillow,
  aspect-ratio preserved with transparent letterboxing).
- Upload images to **Supabase Storage** and save the post to **Supabase
  Postgres**, returning the published URL.
- Accept both explicit commands and **natural-language** instructions.
- Print a live **Rich** terminal UI with per-agent progress.

Agents: `ProjectManager` (orchestrator) → `DocumentScanner`, `Extracting`,
`ImageProcessing`, `Uploading`, `Logging`.

## Prerequisites

- Python 3.10+ and [`uv`](https://docs.astral.sh/uv/).
- For the default `local` provider: [Ollama](https://ollama.com) running with the model.

    ```sh
    ollama serve
    ollama pull qwen3:8b
    ```

- Copy `.env.example` → `.env` and fill in: `LLM_PROVIDER`, `OLLAMA_*` (or
  `OPENAI_*` / `ANTHROPIC_*`), `SUPABASE_URL`, `SUPABASE_SECRET_KEY`,
  `SUPABASE_STORAGE_BUCKET`, `POSTGRES_URL*`.

## Run

```sh
uv sync                                 # install deps
uv run python cli_multi_agent.py        # start the interactive CLI
# or, via the console script:
uv run blog-agent
```

### Inside the CLI (`blog-agent>` prompt)

```text
upload  ./posts/my-article.md     # extract + optimize + upload (publish)
process ./posts/article.md        # extract with metadata generation
analyze ./posts/draft.md          # analyze only, no upload
status | agents | history | help  # system commands
exit
```

Natural language also works, e.g. `Please upload ./posts/new-post.md`.

### Quality gates

```sh
uv run pytest                                   # tests (coverage gate: 80%)
uv run ruff check .                             # lint
uv run mypy agents configs db utils             # type-check
```

## Markdown format

```markdown
---
title: 'Your Post Title'
userId: 1
status: 'public' # 'public' | 'private' | 'follower'
author: 'Your Name'
date: '2026-01-03'
---

# Your Post Title

Content...

![Image](./images/diagram.png)
```

`status` controls visibility (`public` everyone / `private` author-only /
`follower` followers-only). Tags and description are always generated by the LLM.
