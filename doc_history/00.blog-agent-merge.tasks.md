# Blog Agent 병합 — 원자 태스크 DAG

**작성일:** 2026-06-25
**문서 소유자:** Product Manager (PM 에이전트)
**실행 브랜치:** `feature/blog-agent-merge`

> Phase A = 이번 세션(lift-and-shift). Phase B = 별도 세션(Supabase repoint, USER 선행 필요).
> 태스크는 순서를 지켜 실행. 병렬 가능 태스크는 명시.

## 의존성 그래프 (DAG)

```
BA-00
  └── BA-01
        ├── BA-02 ──┐
        └── BA-03 ──┤
                    └── BA-04
                          ├── BA-05 ──┐
                          └── BA-06 ──┤
                                      └── BA-07
                                            └── BA-08
                                                  └── BA-09 (Phase B, 별도 세션)
```

병렬 실행 가능:
- BA-02 와 BA-03 (BA-01 완료 후)
- BA-05 와 BA-06 (BA-04 완료 후)
