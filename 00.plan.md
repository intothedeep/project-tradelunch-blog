### Phase 4 â€” Package Manager Migration (pnpm 9.5.0) (PLANNED)

**Rationale:** yarn 1.22.22 (classic) has no built-in workspace isolation. Phantom dependencies (transitive packages reachable without being declared) are a structural risk. pnpm 9.5.0 with strict isolated `node_modules` eliminates phantom deps, aligns with Turborepo's recommended toolchain, and produces a deterministic lockfile (`pnpm-lock.yaml`).

**Locked decisions (not negotiable):**

- Strict pnpm: `node-linker=hoisted` is forbidden. `shamefully-hoist=true` is forbidden. Phantom-dep breakage must be fixed at the source (add the missing dep to the correct `package.json`).
- pnpm version pinned to `9.5.0` in the `packageManager` field of root `package.json`.
- Workspace declaration moves from `workspaces` field in root `package.json` to `pnpm-workspace.yaml`.
- All documentation files that reference yarn must be updated to reference pnpm equivalents.
- The user runs `pnpm install` manually after all config edits are complete. Engineer agents must NOT run install or delete `node_modules`.

**Scope (all files known to reference yarn):**

| File | Change type |
| --- | --- |
| `package.json` (root) | `packageManager` field, remove `workspaces` field, fix broken script targets |
| `pnpm-workspace.yaml` | Create (new file) |
| `.npmrc` | Create with `strict-peer-dependencies=false` if needed |
| `apps/dashboard_client_web/package.json` | Remove `yarn` from build script |
| `packages/markdown-parsing/package.json` | Remove `yarn run clean &&` from build, deprecated:dev, prepublishOnly scripts |
| `packages/axios/package.json` | Remove `yarn run clean &&` from build script |
| `CLAUDE.md` (root) | Update yarn references |
| `.claude/settings.local.json` | Replace `Bash(yarn install:*)` and `Bash(yarn build)` permissions |
| `apps/dashboard_client_web/README.md` | Update yarn references |
| `apps/docs/README.md` | Update yarn references |
| `packages/markdown-parsing/README.md` | Update yarn references |
| `apps/dashboard_server/docs/node.server.md` | Update yarn references |
| `docs/guide.spec.md` | Update yarn references |

**Acceptance criteria (phase complete when ALL pass):**

- `package.json` root has `"packageManager": "pnpm@9.5.0"` and no `"workspaces"` field
- `pnpm-workspace.yaml` exists at repo root and lists all workspace globs
- No file in the repo contains a bare `yarn` command in any `scripts` field
- `.claude/settings.local.json` contains `Bash(pnpm install:*)` and `Bash(pnpm build)` permissions instead of yarn equivalents
