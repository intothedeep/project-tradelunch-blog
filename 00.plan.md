## Product Identity

Personal portfolio + fintech site for Taek Lim (GitHub: tradelunch), one Next.js 16 monorepo,
domain `my.prettylog.com`. Two surfaces:

1. **Blog / portfolio** ‚Äî curated single-author technical posts, read-only. Posts are authored
   by the Python publishing agent (`apps/blog_agent`); no in-app authoring UI.
2. **Financial dashboard** ‚Äî TradingView-style chart + 22-item watchlist with indicators,
   drawing tools, intraday intervals. Currently mock data; Cycle 3 wires the real backend.

## Architecture Decisions (resolved)

**`@repo/types` package (2026-06-24).** Lean package (no build step, raw TS, `main`/`types` ‚Üí
`src/index.ts`) for API-boundary types shared front+back: `ETreeNodeType`, `TCategoryTreeNode`,
`TPostTreeNode`, `TCategoryTreeResponse`, `TTreeNode`, `TTreeNodeWithChildren`. Client adds
`transpilePackages: ['@repo/types']`. `@repo/markdown-parsing` later retired from the client
(TOC inlined to `utils/markdown/`).

**Env var convention (kcdd2).** SCREAMING_SNAKE_CASE, no prefix namespacing (`DATABASE_URL`,
not `SUPABASE_DATABASE_URL`). Key vars: `DATABASE_URL` (Supabase pooler 6543), `NODE_ENV`,
`SITE_DOMAIN`, `API_SITE_DOMAIN`, `CDN_ASSET_POSTS`.

**Data access.** Raw SQL kept verbatim. Sequelize execution layer replaced by `pg` Pool
(`max: 1`) ‚Üí Supabase transaction pooler (port 6543). No supabase-js. The 591-line `posts.ts`
SQL is unchanged.

**Backend topology.** `dashboard_server` stays in the monorepo at `apps/dashboard_server`.
Two Vercel projects, one repo. Vercel Root Directory = the app dir; install/build from repo root.

**blog_agent merge (2026-06-25).** The Python LangGraph publishing agent was merged into the
monorepo as `apps/blog_agent` (lift-and-shift, code logic unchanged). Turbo recognises it via a
thin `package.json` stub delegating to `uv run`; `start` is excluded from `dev`/`build`, only
`test`/`lint`/`check-types` are filterable. Schema SSOT is
`apps/dashboard_server/schema/tradelunch.schema.sql`; `apps/blog_agent/schema/‚Ä¶` is a byte-identical
copy guarded by `pnpm check:schema` (drift detection). See "blog_agent" under Phase A/B below.
Full ADR archived in `_docs/00.blog-agent-merge.arch.md`.

### Phase A ‚Äî Deploy Current State (Free, Read-Only)

**Goal:** current version ‚Äî read-only blog + mock dashboard ‚Äî live on Vercel at zero cost, zero
new features.

**Done means:** frontend live on Vercel (later `my.prettylog.com`); backend live as a second
Vercel project reading Supabase via `pg` Pool (raw SQL, no SSH tunnel, no Sequelize); dashboard
still mock; CI runs typecheck on every push; auto-deploy on merge to main.

**blog_agent (Phase A merge ‚Äî DONE, commit f51cc7e):** Python agent lifted into
`apps/blog_agent`, turbo-integrated, schema-drift guard added. Runtime infra untouched (still
`DB_PG_*`/`AWS_*` env contract) ‚Äî repoint is Phase B.

**Acceptance gate (all must hold before Phase B):**
- `https://my.prettylog.com/blog/@taeklim` loads posts from Supabase; infinite scroll works; no SSH/AWS
- `https://my.prettylog.com/dashboard` loads chart with mock data; all 22 watchlist items visible
- CI green on main
- No hardcoded secrets in source
- Both apps `check-types` exit 0; no `sequelize` / `supabase-js` import under `dashboard_server`

**Non-goals:** real market data, auth, any new feature, storage migration, moving server out of monorepo.

### Phase B ‚Äî DB/Storage Repoint + Real Dashboard Data

**Goal:** retire AWS Postgres + SSH tunnel + S3; serve real market data; repoint blog_agent to
Supabase/Cloudflare.

**Done means:** server reads all data from Supabase via `pg` Pool; Cycle 3 Server Actions switch
mock‚Üíbackend on `DASHBOARD_DATA_SOURCE=backend`; all 22 items show real values + 1-day change;
blog images served from Supabase Storage `blog-images`; AWS S3 SDK/env vars removed.

**blog_agent (Phase B repoint ‚Äî BA-09, separate session, USER prereqs):** `DB_PG_*` ‚Üí
`DATABASE_URL` (asyncpg + Supabase pooler); boto3/AWS S3 ‚Üí Supabase Storage S3-compatible
endpoint + `blog-images` bucket; `CDN_ASSET_POSTS` ‚Üí Cloudflare CNAME; clean RDS CA / SSH /
boto3 remnants. Ties off `00.tasks.md` S8.3.
- **USER prereqs:** create `blog-images` bucket; set Cloudflare CNAME; confirm pooler `DATABASE_URL`.

**Storage decision:** AWS S3 deleted ‚Üí Supabase Storage is a fresh setup (permanent 1 GB free
tier), not a migration. `CDN_ASSET_POSTS` points at the Supabase Storage public URL, optionally
fronted by a Cloudflare CNAME later (name/contract unchanged either way).

**Ingestion cron:** DEFERRED (post-refactor). Preferred future host: GitHub Actions scheduled
workflow (avoids Vercel 10s timeout). Supabase 7-day auto-pause is an accepted known risk;
fallback = manual resume.

**Non-goals:** auth, user-facing writes, ingestion cron.

### Phase D ‚Äî Multi-User Blog Platform (Invite-Only)

Largest phase; do not begin until Phase B is stable. Signup is **invite-only** (admin-issued
codes in an `invites` table); open public signup is out of scope.

- **D1 ‚Äî Auth foundation:** Clerk + Supabase JWT bridge; `clerk_user_id()` SQL helper; `users` +
  `invites` tables; `user.created` webhook validates invite before insert; RLS `owner_id` /
  `published`; protected-route middleware.
- **D2 ‚Äî Authoring UI:** `/write` Markdown editor; draft auto-save (`published=false`);
  `POST`/`PATCH`/`DELETE /api/posts` owner-scoped; image upload to Supabase Storage.
- **D3 ‚Äî Publish + profiles:** publish/unpublish; `/blog/@[username]` + `/[slug]`; author byline;
  `/settings/profile`.
- **D4 ‚Äî Admin moderation:** `is_admin`; `/admin/posts` (unpublish/hard-delete); `/admin/invites`
  (generate/revoke); admin-only endpoints via RLS + Clerk role check.
