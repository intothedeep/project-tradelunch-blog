      inspect `x-vercel-cache: HIT/MISS` + `age`. MISS-on-repeat ⇒ `s-maxage` not honored
      (fix caching first — may erase the bill with zero migration).
- [ ] **0.3 Cap the two unbounded queries.** Add sane `LIMIT`/clamp to
      `controllers/funds/funds.ts:195` (holdings) and `controllers/dashboard/dashboard.ts:140`
      (snapshot). Re-measure egress. (Independent quick win; do regardless of the move.)
- [ ] **0.4 Provision Oracle VM + PG17.** TLS on, `5432` open, stable public IPv4,
      `pg_hba.conf`/security-lists locked to TLS-only. Confirm `psql -c 'select 1'` reaches
      it from **both** a Vercel region **and** GitHub runners. Confirm the VM is a
      paid/reserved shape (not idle-reclaimable Always Free).
- [ ] **0.5 Install PgBouncer/pgcat** (transaction pooling) on the VM. Required for
      serverless burst safety even though `finance_api` is co-located.
- [ ] **0.6 Backups.** Configure `pgBackRest`/WAL archiving → off-box/object storage +
      a **tested restore** (untested backup ≠ backup). Financial history is expensive to re-collect.
- [ ] **0.7 Generate financial DDL.** `pg_dump --schema-only --no-owner --no-privileges
    -t <financial tables>` → `packages/finance-migrations/0001_finance_init.sql`. Strip
      Supabase-isms (`auth.`/`storage.`/RLS/`supabase_*`). Review by hand.
- [ ] **0.8 Apply DDL to Oracle.** `psql -v ON_ERROR_STOP=1 -f 0001_finance_init.sql`;
      `REFRESH MATERIALIZED VIEW mv_sec_new_positions;`. Verify with `\dt` + smoke selects.
      _Rollback: none — Oracle is greenfield._
